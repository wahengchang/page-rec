# Extension code rules

The v2.0 design system in @DESIGN.md is the single source of truth for all UI/UX decisions in `extension/sidepanel.{html,css,js}`. Read DESIGN.md before making any visual change.

## Design tokens (hard rules)

- **Colors are CSS variables** in `extension/sidepanel.css` under `:root`. Never hardcode hex codes — use `var(--color-primary)`, `var(--text-secondary)`, etc.
- **Light + dark mode swap automatically** via `@media (prefers-color-scheme: dark)`. When adding a new color variable, add a dark-mode counterpart in the same PR.
- **Max 4px border-radius on buttons. 0px on containers/panels.**

## Typography

- **UI text**: `var(--font-ui)` (Inter + system fallbacks) at 13px base.
- **Data / selectors / timings**: `var(--font-mono)` (JetBrains Mono + fallbacks) at 12px.
- **Weight 500** for verbs/actions, **400** for data strings.

## List items (Timeline + Step editor)

- **Max 2-line clamp** via `-webkit-line-clamp: 2` on text containers.
- **Native tooltip** (`title` attribute) on `<li>` or wrapper must contain the full unclamped text so users can hover to read.
- **Padding**: 6-8px vertical, 12px horizontal (high-density per SSOT).
- **Flat rows, no cards** — single `1px solid var(--border-divider)` separator, no shadows.

## Verb color coding

In `wrapVerbHtml` (`extension/sidepanel.js`), the leading action word gets a `.verb verb-*` class:

| Verb | Class | Color |
|---|---|---|
| Clicked | `verb-click` | `var(--color-primary)` (Indigo) |
| Typed | `verb-type` | `var(--color-input)` (Amber) |
| Navigated | `verb-nav` | `var(--color-success)` (Emerald) |
| Submitted | `verb-click` | Indigo |
| Scrolled | `verb-scroll` | `var(--text-secondary)` (Slate) |

When adding a new action type, add a corresponding entry + CSS class.

## Sensitive files

- **`content.js`** — pre-existing `__rec=` dead code is intentionally kept (dedicated cleanup later). Do not edit this file as a side-effect of other work.
- **`background.js`** — the `chrome.tabs.onUpdated` listener is narrowed to re-injection only (see @docs/ARCHITECTURE.md § Content script lifecycle). Do **not** delete this listener; removing it silently breaks multi-page recording. The re-injection branch must **not** write to `chrome.storage.session.config`.
- **`export.js`** — `window.generateTest` is called from `sidepanel.js`. If you change its signature, update all callers in the same commit.
- **`manifest.json` `host_permissions`** — `http://*/*` and `https://*/*` are required for recording on any site. Do not trim without discussion. Localhost (`http://127.0.0.1/*`) was intentionally removed during the standalone refactor — do not add it back.

## Dependency policy

**Zero npm dependencies** in the extension. All code must run in the Chrome MV3 runtime without a bundler. This is a Chrome Web Store submission requirement — do not import from `node_modules`.
